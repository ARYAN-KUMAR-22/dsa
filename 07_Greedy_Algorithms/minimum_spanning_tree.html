Problem Statement

Given a connected, undirected graph with weighted edges, find a subset of edges that:

  • Connects all vertices (forms a tree)
  • Uses N-1 edges for N vertices
  • Has minimum total weight among all possible spanning trees
  • Has no cycles

Key Applications

  • Network Design: Minimize cost of connecting cities/servers
  • Electrical Grids: Minimize wiring between power stations
  • Telecommunications: Optimal phone network layout
  • Robotics: Efficient movement path planning
  • Image Processing: Region clustering and merge operations

Greedy Approach

MST algorithms use the greedy strategy:

  • Kruskal's: Sort edges by weight, add minimum weight edge if it doesn't create cycle
  • Prim's: Start from a vertex, repeatedly add minimum weight edge connecting visited to unvisited vertices
  • Both are optimal due to the "cut property" of graphs

Why Greedy Works (Cut Property)

For any cut of the graph, the minimum weight edge crossing the cut is part of some MST.

  • At each step, we partition vertices into two sets
  • Taking the minimum edge between sets maintains optimality
  • Proof by exchange argument (cannot replace with smaller edge)

Kruskal's Algorithm

1. Sort all edges by weight (ascending)
2. Initialize empty MST
3. For each edge in sorted order:
4.    If edge connects two different components:
5.        Add edge to MST
6.        Merge the two components
7. Return MST (N-1 edges)

Time: O(E log E) for sorting
Space: O(V) for union-find

Prim's Algorithm

1. Start with arbitrary vertex
2. Initialize MST with this vertex
3. While vertices remain unvisited:
4.    Find minimum weight edge from MST to unvisited vertex
5.    Add this edge and vertex to MST
6. Return MST (N-1 edges)

Time: O(E log V) with binary heap
Space: O(V + E)

Example: Network Design

Problem Setup:

Vertices: A, B, C, D, E (5 cities)

Edges: A-B(4), A-C(2), B-C(1), B-D(5), C-D(8), D-E(3)

Kruskal's Solution:

Step 1: Sort edges: B-C(1), A-C(2), D-E(3), A-B(4), B-D(5), C-D(8)

Step 2: Add B-C(1) ✓ [Components: {B,C}, {A}, {D}, {E}]

Step 3: Add A-C(2) ✓ [Components: {A,B,C}, {D}, {E}]

Step 4: Add D-E(3) ✓ [Components: {A,B,C}, {D,E}]

Step 5: Add B-D(5) ✓ [Components: {A,B,C,D,E}] - MST Complete!

Result:

MST Edges: B-C(1), A-C(2), D-E(3), B-D(5)

Total Cost: 1 + 2 + 3 + 5 = 11

All vertices connected: ✓

Example: Prim's Algorithm

Using same graph, starting from A:

Start: MST = {A}

Iter 1: Min from A: A-C(2) → MST = {A,C}

Iter 2: Min from {A,C}: B-C(1) → MST = {A,B,C}

Iter 3: Min from {A,B,C}: B-D(5) → MST = {A,B,C,D}

Iter 4: Min from {A,B,C,D}: D-E(3) → MST = {A,B,C,D,E}

Same Result: Total cost = 11

Advantages

  • ✓ Always produces optimal solution (mathematically proven)
  • ✓ Efficient: Kruskal O(E log E), Prim O(E log V)
  • ✓ Works on any connected undirected weighted graph
  • ✓ Intuitive greedy approach
  • ✓ Well-studied with many implementations

Disadvantages

  • ✗ Requires graph to be connected
  • ✗ Prim requires dense graph for efficiency
  • ✗ Kruskal requires sorting (O(E log E))
  • ✗ Not applicable to directed graphs (use different algorithms)

Real-World Use Cases

  • Electricity Distribution: Minimize cable length connecting households
  • Road Networks: Optimal highway construction between cities
  • Airlines: Minimum distance to connect all airports
  • Internet Backbones: Efficient server connectivity
  • Data Clustering: Single-link agglomerative clustering

Key Formulas

Number of edges in MST: N - 1 (for N vertices)
Kruskal Time: O(E log E + α(V)) where α is inverse Ackermann
Prim Time: O((V + E) log V) with binary heap
Space: O(V + E) for graph representation

Important Properties

  • Cut Property: Min weight edge across cut is in some MST
  • Cycle Property: Max weight edge in any cycle is not in MST
  • Uniqueness: If all weights distinct, unique MST exists
  • Exchange Property: Can transform one MST to another by edge exchange
