Definition & Core Concept

An optimization problem is a mathematical problem where we seek to find the best solution from all feasible solutions. The goal is to either maximize or minimize an objective function while satisfying a set of constraints.

An optimization problem consists of:

  • Objective Function: A function f(x) that we want to maximize or minimize
  • Decision Variables: The variables we can control (represented as x)
  • Constraints: Limitations or restrictions that the solution must satisfy
  • Feasible Region: The set of all solutions that satisfy the constraints
  • Optimal Solution: The best feasible solution that optimizes the objective function
Example: Minimize cost while delivering packages on time following truck capacity limits.

Types of Optimization Problems

Linear Programming

Both objective function and constraints are linear equations/inequalities.

Non-Linear Programming

Objective function or constraints contain non-linear terms.

Integer Programming

Decision variables must be integers. Useful for discrete choices.

Combinatorial Optimization

Finding optimal solution from a finite set of possible solutions.

Convex Optimization

Objective and constraint functions are convex. Guarantees global optimum.

Non-Convex Optimization

May have multiple local optima. Finding global optimum is harder.

Maximization vs Minimization

  • Maximization: Find the highest value (e.g., maximize profit, efficiency)
  • Minimization: Find the lowest value (e.g., minimize cost, time, distance)
  • Any maximization problem can be converted to a minimization problem by negating the objective function

Key Characteristics

  • Feasibility: A problem is feasible if at least one solution satisfies all constraints
  • Boundedness: A minimization problem is unbounded if the objective can decrease indefinitely
  • Uniqueness: An optimal solution may be unique or there may be multiple optimal solutions
  • Sensitivity: How much the optimal solution changes when problem parameters change

Solution Approaches

Different approaches exist depending on problem complexity:

  • Exact Algorithms: Guarantee optimal solution (may take exponential time for large instances)
  • Greedy Algorithms: Make locally optimal choices (fast but not always optimal)
  • Dynamic Programming: Break problem into overlapping subproblems
  • Heuristics & Metaheuristics: Approximation techniques like simulated annealing, genetic algorithms
  • Approximation Algorithms: Find near-optimal solutions with proven quality guarantees

Optimization Problems & Greedy Algorithms

Greedy algorithms are a popular approach for solving optimization problems. A greedy algorithm works by:

  1. Making the locally optimal choice at each step
  2. Hoping that this leads to a globally optimal solution
  3. Never reconsidering previous choices

Greedy algorithms are fast and simple but don't always produce optimal solutions. They work well for problems with greedy choice property and optimal substructure.

Example: The activity selection problem uses a greedy approach: always select the activity that finishes earliest. This maximizes the number of non-overlapping activities.

NP-Hardness

Some optimization problems are classified as NP-Hard:

  • No known polynomial-time algorithm exists to find the optimal solution
  • Verification of a solution can be done in polynomial time
  • Examples: Traveling Salesman Problem, Knapsack Problem, Vertex Cover
  • For NP-Hard problems, use approximation algorithms or heuristics instead of exact algorithms

Real-World Examples

  • Knapsack Problem: Maximize total value while keeping total weight within a limit
  • Traveling Salesman Problem (TSP): Find the shortest route visiting all cities exactly once
  • Activity Selection: Select maximum number of non-overlapping activities
  • Scheduling: Minimize task completion time or machine setup costs
  • Portfolio Optimization: Maximize returns while minimizing investment risk
  • Network Flow: Maximize flow through a network with capacity constraints
  • Resource Allocation: Distribute limited resources to maximize total benefit
  • Job Sequencing: Schedule jobs to maximize profit before deadlines

Greedy vs Dynamic Programming

Greedy Algorithm

  • Makes local optimal choice
  • Never backtracks
  • Fast (usually O(n) or O(n log n))
  • Not always optimal
  • Good for specific problem types

Dynamic Programming

  • Explores multiple choices
  • Can backtrack via memoization
  • Slower (often O(n²) or more)
  • Always optimal
  • Works for many problem types

Key Takeaways

  • Optimization problems seek the best solution from all feasible solutions
  • They consist of an objective function, variables, and constraints
  • No single approach works for all optimization problems
  • Greedy algorithms are fast but not always optimal
  • Some problems (NP-Hard) require approximation or heuristic approaches
  • Choosing the right algorithm depends on problem size, solution quality, and time constraints

  • Real-valued
  • Mixed (integer+real)

When to Choose Each Approach

  • Greedy: Problem has greedy choice property, fast solution needed, optimal substructure exists
  • Dynamic Programming: Optimal substructure, overlapping subproblems, small to medium problem size
  • Exact Algorithms: Small instance size, can afford exponential time, need guaranteed optimal
  • Approximation: Large NP-Hard instances, need polynomial-time guarantee with quality bound
  • Heuristics: Very large instances, any reasonable solution acceptable, time critical

Optimization in Computer Science

Optimization problems appear everywhere:

  • Machine Learning: Minimizing loss functions, training neural networks
  • Operations Research: Resource allocation, scheduling, routing
  • Database Systems: Query optimization, index selection
  • Compiler Design: Code optimization, register allocation
  • Game Development: Pathfinding, resource management
  • Networking: Routing protocols, load balancing
