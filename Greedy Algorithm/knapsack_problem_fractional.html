Problem Statement

You have a knapsack with a maximum weight capacity of W. There are n items, each with:

  • Weight (wi): How much the item weighs
  • Value (vi): How much the item is worth

Goal: Select items (you can take fractions of items) to maximize total value without exceeding weight capacity W.

Real-World Example: A shopkeeper at a market can fill a bag with items. Some items can be divided (rice, flour), so you can take partial amounts. What combination maximizes value?

Why Fractional vs 0/1 Knapsack?

0/1 Knapsack

  • Either take entire item OR leave it
  • Cannot divide items
  • Greedy FAILS ❌
  • Use Dynamic Programming
  • NP-Complete

Fractional Knapsack

  • Can take ANY fraction of item
  • Items are divisible
  • Greedy WORKS ✓
  • Use Greedy Algorithm
  • Polynomial time

Why Greedy Works for Fractional Knapsack

Fractional knapsack has the greedy choice property:

  • At each step, the greedy choice (pick highest value-to-weight ratio) is optimal
  • If you can take fractions, you can always fill the knapsack completely
  • No waste occurs - you get items with the best value efficiency first
  • Unlike 0/1 knapsack, there's no situation where taking lower-ratio items helps
Key Insight: Since you can take partial items, completely filling the knapsack is always possible. The greedy choice of taking the highest ratio item (and fractional amounts if needed) guarantees the maximum value.

Greedy Strategy

  1. Calculate Ratio: For each item, compute value-to-weight ratio = vi / wi
  2. Sort by Ratio: Arrange items in descending order of ratios
  3. Greedy Selection: Starting from highest ratio:
    • If entire item fits, take it completely
    • If partial item fits, take only what fits
    • Continue until knapsack is full or items exhausted
  4. Result: Maximum value with divisible items

Mathematical Formulation

Maximize:

∑(xi × vi) for i = 1 to n

Subject to:

∑(xi × wi) ≤ W
0 ≤ xi ≤ 1 for all i

Where xi is the fraction of item i to take (0 = nothing, 1 = entire item)

OUTPUT: maximum total value
Line 1: totalValue ← 0
Line 2: totalWeight ← 0
Initialize result variables to zero

ratio[i] ← value[i] / weight[i]
Calculate efficiency (value per kg) for each item
Line 5: Sort items by ratio in DESCENDING order
Highest value-to-weight ratio comes first
Line 6: FOR each item i in sorted order:
Line 7: IF totalWeight + weight[i] ≤ capacity:

// Item doesn't fit completely
remaining ← capacity - totalWeight
fraction ← remaining / weight[i]
totalValue ← totalValue + (fraction × value[i])
totalWeight ← capacity
BREAK
Take partial item and stop (knapsack is full)
Keep selecting items in order until knapsack is full
Line 9: RETURN totalValue
Return the maximum value achieved

Key Points:

  • Lines 1-2: Initialize counters
  • Lines 3-4: Calculate value/weight ratio for efficiency
  • Line 5: Sort by efficiency (best value first)
  • Lines 6-8: Main loop: greedily select items
  • Line 9: Return result

Step-by-Step Process

  1. Input: Array of items with weights and values, capacity W
  2. Calculate Ratios: Efficiency = value ÷ weight for each item
  3. Sort Items: By ratio in descending order (best value first)
  4. Iterate Through Items:
    • Check if item fits completely
    • If yes → Add entire item to knapsack
    • If no → Add only the fraction that fits
    • Stop when knapsack is full
  5. Output: Maximum value achieved

Implementation Considerations

  • Sorting: Must sort items by value-to-weight ratio (descending)
  • Floating Point: Fractions may be decimal values (e.g., 2.5 kg out of 10 kg)
  • Precision: Be careful with floating-point arithmetic edge cases
  • Edge Cases:
    • Empty items list
    • Capacity = 0
    • All items heavier than capacity
    • Items with zero value or weight
  • Comparison with 0/1: Fractional's O(n log n) is much better than 0/1's exponential worst case

Step 1: Calculate Ratios

Item 1: 60 ÷ 10 = 6.0 [TOP] Best
Item 2: 100 ÷ 20 = 5.0
Item 3: 120 ÷ 30 = 4.0

Step 2: Sort by Ratio (Descending)

1. Item 1 (Ratio 6.0)
2. Item 2 (Ratio 5.0)
3. Item 3 (Ratio 4.0)

Step 3: Greedy Selection

Take Item 1 (Entire)
Weight: 10 kg ≤ 50 kg ✓ Take all
Value += 60 | Used: 10 kg | Remaining: 40 kg
Take Item 2 (Entire)
Weight: 20 kg ≤ 40 kg ✓ Take all
Value += 100 (Total: 160) | Used: 30 kg | Remaining: 20 kg
Take Item 3 (Partial)
Weight: 30 kg > 20 kg ✗ Can't take all
Fraction: 20 ÷ 30 = 2/3 of Item 3
Value += (2/3) × 120 = 80 (Total: 240)
Used: 50 kg | Remaining: 0 kg (FULL!)
Final Result:
✓ 10 kg Item 1 (Value: 60)
✓ 20 kg Item 2 (Value: 100)
✓ 20 kg of Item 3 (Value: 80)
Maximum Value: 60 + 100 + 80 = 240
Total Weight: 10 + 20 + 20 = 50 kg (Perfect!)

When to Use Fractional Knapsack

  • Items can be divided or fractioned (liquids, powders, grains)
  • Need optimal solution quickly
  • Time complexity of O(n log n) is acceptable
  • Real-world scenarios with continuous items
  • Compare: 0/1 Knapsack requires DP and is NP-Complete

Real-World Applications

  • Liquid Tank Filling: Fill containers with different liquids to maximize value
  • Resource Allocation: Distribute budget across projects with different ROI
  • Cargo Loading: Load truck with materials that can be partially taken
  • Portfolio Optimization: Allocate money across stocks/bonds with returns
  • Food Production: Mix ingredients with different costs/nutritional values
  • Manufacturing: Allocate materials to maximize production value

Key Takeaways

  • Fractional knapsack is solvable in polynomial time O(n log n)
  • Greedy algorithm works optimally because items are divisible
  • Strategy: Sort by value/weight ratio, then greedily take items in order
  • Always ends with a full knapsack (can take partial last item)
  • Much easier than 0/1 knapsack (which is NP-Complete)
  • Practical applications abound in resource allocation problems

Quick Decision Tree

  1. Can items be divided?
    • Yes → Use Fractional Knapsack (Greedy) ✓
    • No → Use 0/1 Knapsack (DP)
  2. Need guarantee of optimality? → Yes, both methods work
  3. Need fast execution? → Fractional is O(n log n) vs 0/1's O(n×W)
