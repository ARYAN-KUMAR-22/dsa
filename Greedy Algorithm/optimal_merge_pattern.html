Algorithm Theory

Problem Overview

Given k sorted arrays of different sizes, merge them into a single sorted array with minimum total cost. The cost of merging two arrays is the sum of their sizes.

Key Insight

This is a greedy problem. At each step, we should merge the two smallest arrays first. This minimizes the total cost because:

  • Smaller arrays contribute less to subsequent merge operations
  • Merging small arrays early prevents them from being merged multiple times
  • This greedy choice leads to the optimal solution (proven by exchange argument)

Greedy Strategy

Always merge the two smallest remaining arrays. This ensures that:

  • Small merged results are handled efficiently in future merges
  • We minimize the sum of all intermediate merge costs
  • No array is unnecessarily merged multiple times with large arrays

Time Complexity

O(k² log k) where k is the number of arrays

  • We perform k-1 merge operations (each reduces array count by 1)
  • Each merge requires finding the 2 smallest: O(log k) with min-heap
  • Total: O(k log k) for heap operations + O(k²) for copying elements

Space Complexity

O(k) for the min-heap storing array sizes

Example Trace

Arrays: [4, 3, 2, 6, 5]

  1. Min-heap: [2, 3, 4, 5, 6] → Merge 2+3=5 → Heap: [4, 5, 5, 6], Cost: 5
  2. Merge 4+5=9 → Heap: [5, 6, 9], Cost: 9+5=14
  3. Merge 5+6=11 → Heap: [9, 11], Cost: 14+11=25
  4. Merge 9+11=20 → Heap: [20], Cost: 25+20=45

Total Cost: 45

Pseudocode
function OptimalMergePattern(arrays[]):
    minHeap = CreateMinHeap()
    totalCost = 0
    
    // Insert all array sizes into min-heap
    for each size in arrays:
        minHeap.insert(size)
    
    // Merge until one array remains
    while minHeap.size() > 1:
        // Extract two smallest arrays
        first = minHeap.extractMin()
        second = minHeap.extractMin()
        
        // Calculate cost of merging
        cost = first + second
        totalCost += cost
        
        // Insert merged array back
        minHeap.insert(cost)
        
        // Log the operation
        log("Merged [" + first + "] + [" + second + "] = [" + cost + "]")
    
    return totalCost

Implementation Details

Data Structure: Min-Heap

  • Efficiently retrieves two smallest elements: O(log k)
  • Insert merged result back: O(log k)
  • Alternative: Use priority queue

Alternative Approach:

  • Sort array: O(k log k)
  • Repeatedly extract from sorted list: O(k²)
  • Less efficient than min-heap approach
Key Concepts

🎯 Greedy Algorithm

An algorithm that makes locally optimal choices hoping to find a global optimum. In this problem, choosing to merge the two smallest arrays at each step leads to the globally optimal solution.

📚 Min-Heap (Priority Queue)

A complete binary tree where each parent node has a value less than or equal to its children. Provides O(log n) insertion and extraction of minimum.

  • Insert: Add element at end, bubble up
  • ExtractMin: Remove root, replace with last element, bubble down

💰 Cost Function

The cost of merging two sorted arrays is the sum of their sizes. This represents the number of comparisons/operations needed.

Example: Merging array of size 3 with array of size 5 costs 3+5=8

🔄 Huffman Coding Connection

This problem is identical to building an optimal Huffman tree. We're essentially creating a merge tree where leaf nodes represent input arrays and internal nodes represent merge operations.

✅ Optimality Proof Sketch

Exchange Argument:

  • Suppose an optimal solution merges arrays A and B in step i, and later merges C and D in step j (where i < j)
  • If we swap and merge C and D first, then A and B, we can show the total cost doesn't increase
  • This means for any optimal solution, merging smallest arrays first is always optimal
