The Problem

Given a sequence of matrices A₁, A₂, ..., Aₙ where matrix Aᵢ has dimensions pᵢ₋₁ × pᵢ, find the most efficient way to multiply them all together.

Key insight: Matrix multiplication is associative, so (AB)C = A(BC), but the number of scalar multiplications differs!

Example

Matrices: A(10×30), B(30×5), C(5×60)

(AB)C: 10×30×5 + 10×5×60 = 1500 + 3000 = 4500 mults
A(BC): 30×5×60 + 10×30×60 = 9000 + 18000 = 27000 mults

→ First way is 6× better!

Recurrence Relation

m[i,j] = minimum cost to multiply matrices i through j
m[i,i] = 0  (single matrix, no multiplication)

For i < j:
  m[i,j] = min { m[i,k] + m[k+1,j] + p[i-1] × p[k] × p[j] }
           for all i ≤ k < j

Where p[i-1] × p[k] × p[j] is the cost of multiplying
the two subproducts (Aᵢ...Aₖ) and (Aₖ₊₁...Aⱼ).

DP Table Structure

Rows: Starting matrix index
Columns: Ending matrix index
Cell m[i][j]: Minimum multiplications to compute matrices i through j

Fill order: By chain length (diagonal by diagonal)
- Length 1: m[1,1], m[2,2], etc. (all 0)
- Length 2: m[1,2], m[2,3], etc.
- Length n: m[1,n] (final answer)

Time & Space Complexity

Time: O(n³) - three nested loops
Space: O(n²) - two tables (cost + split indices)

This is very efficient! Without DP, there are Cₙ (Catalan number) possible parenthesizations, which grows exponentially.

Reconstruction

During DP computation, store which k gave the minimum.
After filling the table, trace back from s[1,n] to find the split points.
This reveals the optimal parenthesization.
