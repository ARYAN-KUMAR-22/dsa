Memoization vs. Pure Recursion

          

            
Approach
Pure recursion: Re-computes the same states many times.
Memoization: After first computation, all repeated states return instantly from cache.

            
Function calls

            
Time complexity
Pure: O(2^n) in worst case due to exponential branching.
Memo: O(n × w) because each unique state is solved once.

            
When to use
Memoization: Top-down: start with the full problem, break into subproblems, cache results.
Tabulation: Bottom-up: fill a table iteratively from base cases up.

            
Space tradeoff
Pure: O(depth) call stack only.
Memo: O(n × w) for cache dictionary + O(depth) for stack.

            
Clarity
Pure: Shows the original mathematical recurrence cleanly.
Memo: Same logic, but with practical speedup and minimal overhead.

            
Implementation
Pure: Recursion only, no data structures needed.
Memo: Add a dictionary and check before branching.
