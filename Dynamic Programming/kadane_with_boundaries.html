Enhanced Kadane's with Boundaries

Kadane's algorithm finds the maximum sum in O(n), but what if you also need to know which elements form that subarray? This version tracks start and end indices!

Key Additions

Track 4 pointers:
• currentStart: where current sum began
• maxStart, maxEnd: boundaries of best subarray
• Update boundaries when new max found

Logic Flow

1. If current_sum becomes 0 → reset currentStart
2. If current_sum exceeds max → update max AND update boundaries
3. Display exactly which elements form maximum
4. Also show the actual subarray sum

Practical Example

Array: [-2, 1, -3, 4, -1, 2, 1, -5, 4]
Max Sum: 6
Subarray: [4, -1, 2, 1] (indices 3-6)
This visualization shows elements 3,4,5,6 highlighted

Why This Matters

• Interview question: often asks for boundaries too
• Real-world: need to know WHICH elements, not just sum
• Extends algorithm: minimal overhead for big benefit
• Still O(n) time, O(1) space!
