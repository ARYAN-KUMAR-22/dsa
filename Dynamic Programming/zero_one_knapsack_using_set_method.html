Common Mistakes

          

            
Skipping sort before prune
If pairs are not processed in weight order, the domination check becomes unreliable.

            
Keeping overweight pairs
The set should drop pairs whose weight already exceeds the knapsack capacity.

            
Wrong domination test
A pair is dominated when another pair has lower or equal weight and greater or equal value, not just when values differ.

            
Assuming it is greedy
The set method is still an exact dynamic programming technique. It does not rely on a local selection rule.

            
Losing reconstruction info
If you store only weights and values, you may lose track of which items created the optimal pair.

            
Expecting a tiny frontier always
Pruning often shrinks the state space a lot, but worst-case growth can still be significant.

          

        


        

          
Quick Reference

          

            
State set
Keep a set of reachable (weight, value) pairs after each processed item.

            
Transition
Create shifted pairs by adding the current item to every existing pair, then merge them back.

            
Pruning
After sorting by weight, keep only pairs whose value is strictly better than every lighter pair seen so far.

            
When it shines
Useful when the reachable frontier stays much smaller than the full (n+1)(W+1) table.
