Why This Problem Matters

          

            
Greedy temptation
Sorting by value, weight, or ratio feels natural, but 0/1 constraints mean a locally good pick can block a better pair later.

            
State design
The pair (i, w) is enough because after choosing among the first i items, only remaining capacity still matters.

            
Why recursion repeats
Recursive take-or-skip branches often revisit the same item index and remaining capacity from different paths.

            
Why 1D DP works
When processing one item at a time, backward capacity iteration preserves the previous row logically without storing it explicitly.

          

        

      


      

        

          
Common Mistakes

          

            
Using greedy ratio
That rule solves fractional knapsack, not 0/1 knapsack. Whole-item constraints change the answer space.

            
Wrong state meaning
If you mix 'remaining items' and 'processed items' inconsistently, the recurrence becomes hard to trust.

            
Forward iteration in 1D
For 0/1 knapsack, iterating capacity upward can accidentally reuse the same item multiple times.

            
Forgetting reconstruction
The maximum value alone is not always enough. Many problems also want the selected items.

            
Huge capacity tables
If capacity is very large, classic O(nW) DP may be too expensive and another formulation may be needed.

            
Ignoring the contrast
Always say why this is not fractional knapsack. That contrast is often the key teaching point.

          

        


        

          
Quick Reference

          

            
State
dp[i][w] = best value using first i items with capacity w.

            
Transition
max(dp[i-1][w], value[i] + dp[i-1][w-weight[i]]) when the item fits.

            
Time
O(nW) because each item-capacity pair is evaluated once.

            
Space
O(nW) for the full table or O(W) with backward iteration.
