Understanding LCS Tabulation

Bottom-Up DP: Build the solution from smallest subproblems to largest. Start with base cases (empty strings = 0 match), then progressively solve larger subproblems.

Step-by-Step Logic

For each cell [i][j]:
• Compare str1[i-1] with str2[j-1]
• If MATCH: Take diagonal value + 1 (we found another char)
• If different: Take max(left, top) (best of skipping one string)
• Fill row by row, left to right

Visual Guide

🟦 Header: String characters (for reference)
⬜ Uncomputed: Will be filled
🟩 Green: Successfully computed cells
🟧 Orange: Currently being computed

Why Tabulation?

• Time: O(m×n) — guaranteed, 3 nested loops
• Space: O(m×n) — 2D table
• Advantage: No recursion overhead, iterative & predictable
• vs Recursion: Recursion is O(2^n), memoization has cache overhead

Example: "AB" & "CD"

No common characters → LCS = 0 (result at [2][2])

Try: "AGGTAB" & "GXTXAYB" → LCS = 5 (GTAB)
