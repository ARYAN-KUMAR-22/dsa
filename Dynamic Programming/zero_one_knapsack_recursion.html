Common Mistakes

          

            
Wrong state meaning
solve(i, w) means the best value from item i onward with remaining capacity w, not the whole problem from scratch again.

            
Trying invalid take branches
If an item does not fit, recursion should skip it instead of creating negative-capacity calls.

            
Missing base cases
When i == n or w == 0, return 0 immediately.

            
Ignoring overlap
Repeated states are the reason memoization works. Plain recursion is the explanation, not the efficient endpoint.

            
Losing reconstruction details
The numeric answer is not enough if you also want the chosen items. That information must be tracked separately.

            
Assuming recursion is fast enough
The decision tree can grow exponentially with the number of items even when the number of distinct states is much smaller.
