Dynamic Programming Approach

Define: dp[i] = maximum sum of subarray ending at position i
Recurrence: dp[i] = max(arr[i], dp[i-1] + arr[i])
Answer: max(dp[0], dp[1], ..., dp[n-1])

The Key Insight

For each position, we have two choices:
✓ Start fresh: dp[i] = arr[i]
✓ Extend previous: dp[i] = dp[i-1] + arr[i]
Pick whichever is larger!

Why It's DP

Optimal Substructure: Best subarray ending at i depends on best subarray ending at i-1
Overlapping Subproblems: dp[i] reuses dp[i-1]
Memoization: Store results in dp table

Example Walkthrough

arr = [-2, 1, -3, 4, -1, 2, 1, -5, 4]
dp[0] = -2 (start)
dp[1] = max(1, -2+1) = 1 (start fresh)
dp[2] = max(-3, 1-3) = -2 (continue)
dp[3] = max(4, -2+4) = 4 (start fresh)
... Result: 6 at [4,-1,2,1]

Complexity Comparison

Brute Force: O(n²) — all subarrays
Kadane's (Optimized DP): O(n) time, O(1) space
DP (This): O(n) time, O(n) space (explicit table)
