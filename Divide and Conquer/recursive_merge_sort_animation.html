Common Mistakes

          

            
Wrong base case
If the recursion does not stop at single-element ranges, the call tree never bottoms out correctly.

            
Broken copy-back
After filling the temporary buffer for one range, copy only that merged segment back to the main array.

            
Losing stability
When equal values appear, taking from the left half first preserves stable ordering exactly as merge sort promises.

            
Range confusion
Keep the recursive subranges and the merge window straight. The midpoint splits calls first, then the merge rebuilds that exact range.
