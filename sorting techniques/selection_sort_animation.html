Complexity

Best: O(n^2) Average: O(n^2) Worst: O(n^2) Space: O(1) Stable: No

Current Insight

Selection Sort repeatedly selects the smallest remaining element and swaps it into the next sorted position.

How Selection Sort Works

  • The left part of the array becomes the sorted prefix one position at a time.
  • For each pass, scan the remaining unsorted suffix to find the minimum value.
  • Swap that minimum into the first unsorted position.

Pseudo Logic

  • for i = 0 to n - 2
  • minIndex = i
  • for j = i + 1 to n - 1
  • if a[j] < a[minIndex]
  • minIndex = j
  • if minIndex != i then swap

When It Is Useful

  • Useful when minimizing writes matters more than minimizing comparisons.
  • Easy to implement and reason about for teaching and debugging.
  • Per pass, it performs at most one swap after finding the minimum.

Key Tradeoffs

  • Its best, average, and worst cases are all O(n^2).
  • The standard version is not stable because swaps can reorder equal values.
  • It sorts in place and uses only O(1) extra space.

Selection Sort At a Glance

  • Selection Sort is in place and simple, but it is not stable in its basic form.
  • Its time stays O(n^2) in the best, average, and worst cases.
  • Its main advantage is that it performs only one swap per pass after finding the minimum.

Compared With Bubble and Insertion

  • Selection Sort usually makes fewer writes than Bubble Sort or Insertion Sort.
  • Insertion Sort is generally faster on nearly sorted arrays because it is adaptive.
  • Bubble Sort is stable, while the standard Selection Sort implementation is not.

Compared With Merge and Quick

  • Merge Sort and Quick Sort are much better for medium and large arrays.
  • Selection Sort keeps its simple control flow, but it gives up too much time as input size grows.
  • Its niche is mostly educational or write-limited situations.

Best Fit

  • Choose Selection Sort when minimizing swaps matters more than minimizing comparisons.
  • It is reasonable for tiny arrays and good for understanding the idea of choosing the next minimum.
  • It is rarely the right answer for high-performance sorting.
