Complexity

Current Insight

Insertion Sort grows a sorted prefix by taking one key at a time and inserting it into the correct position.

How Insertion Sort Works

  • The left part of the array is treated as a sorted prefix.
  • The next unsorted element becomes the key to insert.
  • Larger elements shift right until the key can be placed.

Pseudo Logic

  • for i = 1 to n - 1
  • key = a[i]
  • j = i - 1
  • while j >= 0 and a[j] > key
  • a[j + 1] = a[j]
  • a[j + 1] = key

When It Is Useful

  • Very effective on small arrays and arrays that are already nearly sorted.
  • Often used inside faster hybrid algorithms for tiny partitions.
  • Its best case is linear, O(n), when the input is already sorted.

Key Tradeoffs

  • Average and worst-case time are O(n^2).
  • It is stable, so equal values keep their relative order.
  • It works in place and needs only O(1) extra space.

Insertion Sort At a Glance

  • Insertion Sort is stable, in place, and adaptive, with a best case of O(n).
  • Its worst case is still O(n^2), so it is best for small or partially sorted inputs.
  • It works by extending a sorted prefix one element at a time.

Compared With Bubble and Selection

  • Insertion Sort usually beats Bubble Sort on nearly sorted data because elements move directly to their place.
  • It is also adaptive, unlike Selection Sort, which still scans the whole unsorted region every pass.
  • Selection Sort performs fewer writes, but Insertion Sort is often the faster of the three in practice.

Compared With Shell, Merge, and Quick

  • Shell Sort improves on insertion ideas by moving elements farther early with gaps.
  • Merge Sort and Quick Sort scale much better on large arrays.
  • Insertion Sort still remains useful as a tiny-array helper inside faster hybrid sorts.

Best Fit

  • Choose Insertion Sort for very small arrays, almost-sorted arrays, or as a base case inside larger algorithms.
  • It is a strong educational step up from Bubble Sort because it shows local shifting instead of repeated swapping.
  • For large random arrays, prefer an O(n log n) algorithm.
