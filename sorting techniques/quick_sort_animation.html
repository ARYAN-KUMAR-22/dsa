Current Insight

Quick Sort partitions around a pivot, fixes that pivot, then recursively sorts the left and right subarrays.

How Quick Sort Works

  • Choose a pivot element from the current subarray.
  • Partition values so smaller elements move left of the pivot and larger ones move right.
  • The pivot lands in its final sorted position, then the two sides are sorted recursively.
  • Use the header buttons to compare the recursive call flow with an explicit iterative stack.
  • This visualizer uses the last element of the active subarray as the pivot for Lomuto partitioning.

Pseudo Logic

  • quickSort(low, high)
  • if low >= high return
  • p = partition(low, high)
  • quickSort(low, p - 1)
  • quickSort(p + 1, high)

Partition Pseudo Code

  • partition(low, high)
  • pivot = a[high]
  • i = low - 1
  • for j = low to high - 1
  • if a[j] <= pivot
  • i = i + 1
  • swap(a[i], a[j])
  • swap(a[i + 1], a[high])
  • return i + 1

When It Is Useful

  • Very fast in practice and widely used as a general-purpose in-memory sort.
  • Excellent cache behavior often makes it outperform other O(n log n) sorts.
  • Works especially well when pivot choices avoid highly unbalanced partitions.

Key Tradeoffs

  • Average time is excellent, but poor pivots can degrade it to O(n^2).
  • The standard in-place version is not stable.
  • Recursive calls use stack space, typically O(log n) on balanced partitions.

Quick Sort At a Glance

  • Quick Sort is a fast general-purpose comparison sort with average time around O(n log n).
  • It is usually in-place apart from recursion or stack storage, but it is not stable.
  • Poor pivots can still push it down to O(n^2) in the worst case.

Compared With Merge Sort

  • Quick Sort often wins on in-memory speed and cache behavior.
  • Merge Sort gives guaranteed O(n log n) time and stable ordering.
  • Quick Sort is usually preferred when memory use matters more than stability.

Compared With Bubble, Insertion, and Selection

  • Quick Sort is dramatically better on medium and large arrays than the simple quadratic sorts.
  • The simpler sorts are still easier to teach and sometimes useful for tiny inputs.
  • Quick Sort becomes the practical choice once input size starts to matter.

Best Fit

  • Choose Quick Sort when you want strong average performance on general in-memory arrays.
  • It is excellent when stability is not required and pivot choice is handled well.
  • If guaranteed worst-case behavior matters more, Merge Sort may be safer.
