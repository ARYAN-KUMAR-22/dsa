Current Insight

Counting Sort counts how often each value appears, turns those counts into positions, and writes the output directly.

How Counting Sort Works

  • Create a count array that records how many times each value appears.
  • Convert counts into prefix sums so each value knows its final ending position.
  • Walk the input from right to left to keep equal values in stable order.
  • Place each value directly into the correct output slot, then decrease its count.
  • Copy the output array back to finish with a sorted result.

Pseudo Logic

  • count = array(maxValue + 1, 0)
  • for value in a: count[value]++
  • for i = 1 to maxValue: count[i] += count[i - 1]
  • for i = n - 1 down to 0
  • output[count[a[i]] - 1] = a[i]
  • count[a[i]]--

When It Is Useful

  • Very fast when the key range k is small compared with the number of items n.
  • Useful as a building block inside radix sort and other integer-key algorithms.
  • Great when stable ordering matters and values are small non-negative integers.

Key Tradeoffs

  • It is not comparison-based, so it only works well for restricted integer ranges.
  • Space usage grows with both the input size and the value range.
  • If the maximum value is very large, counting sort can waste memory and lose its advantage.

Counting Sort At a Glance

  • Counting Sort is not comparison-based and runs in O(n + k) when the value range is small.
  • It is stable, but it needs extra memory for both the count array and the output array.
  • Its speed depends less on input order and more on how large the key range is.

Compared With Merge and Quick

  • When k is small, Counting Sort can beat comparison sorts because it does not rely on pairwise ordering tests.
  • Merge Sort and Quick Sort are more flexible because they work for general comparable values.
  • If the range becomes huge, Counting Sort loses its memory and speed advantage quickly.

Compared With Radix Sort

  • Radix Sort uses counting-style passes repeatedly to handle larger integers digit by digit.
  • Counting Sort is simpler when the full value range is already small enough.
  • Radix Sort is more scalable for bounded-width numbers, but Counting Sort is the more direct tool.

Best Fit

  • Choose Counting Sort for small non-negative integer ranges where stability matters.
  • It is especially useful as a building block inside Radix Sort.
  • If the keys are wide, sparse, or not integer-like, use a different family of sort.
