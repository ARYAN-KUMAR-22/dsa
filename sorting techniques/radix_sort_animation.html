Current Insight

Radix Sort repeatedly applies a stable counting pass to each digit place, from least significant to most significant, until the whole array is ordered.

How Radix Sort Works

  • Choose a digit place such as ones, tens, or hundreds.
  • Run a stable counting sort using only that digit from every number.
  • Copy the output back so the array is now sorted by all processed lower digits.
  • Move to the next larger digit place and repeat the same stable pass.
  • Stop after the largest number has no more remaining higher digits.

Pseudo Logic

  • maxValue = max(a)
  • for exp = 1; maxValue / exp > 0; exp *= 10
  • count = array(10, 0)
  • count digits at place exp
  • convert count into prefix sums
  • place values into output from right to left
  • copy output back to a

When It Is Useful

  • Excellent for sorting many non-negative integers when the number of digits d is small.
  • Useful when comparison sorts are not ideal and digit-wise processing is cheap.
  • Works especially well for fixed-width integer keys such as IDs, zip codes, or padded numbers.

Key Tradeoffs

  • It needs a stable inner counting sort for every digit place, so implementation is more involved than a single-pass sort.
  • It is best suited for non-negative integers unless extra handling is added for negatives or strings.
  • If numbers have many digits, the repeated passes can reduce its practical advantage.

Radix Sort At a Glance

  • Radix Sort processes keys digit by digit and can run in about O(d * (n + k)).
  • It is not comparison-based and stays stable when the inner digit sort is stable.
  • Its usefulness depends on bounded digit width and appropriate key formats.

Compared With Counting Sort

  • Counting Sort is simpler when the full key range is already small.
  • Radix Sort extends that idea to larger values by sorting one digit position at a time.
  • If there are many digits, Radix Sort pays for multiple passes.

Compared With Merge and Quick

  • Radix Sort can outperform comparison sorts for fixed-width integers or strings.
  • Merge Sort and Quick Sort are more general because they do not depend on digit structure.
  • Radix Sort wins only when the input format matches its assumptions well.

Best Fit

  • Choose Radix Sort for bounded-width integers, zip codes, IDs, or digit-based keys.
  • It is a strong option when stability matters and non-comparison sorting is possible.
  • If values are arbitrary, comparison-based sorts are usually easier to apply.
