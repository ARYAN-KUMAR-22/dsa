What is STL?

The Standard Template Library (STL) is a generic library providing containers, iterators, and algorithms. It's a fundamental part of C++ that enables efficient data structure usage.

  • Containers: Objects that store collections of data
  • Iterators: Objects that point to elements in containers
  • Algorithms: Functions that operate on containers (sort, find, etc.)

Container Categories

Category Examples Purpose
Sequence vector, deque, list Ordered collection, access by index
Associative map, set, multimap Key-value pairs, sorted access
Adapters stack, queue, priority_queue Restricted access patterns

When to Use Each Container

  • Vector: Most common; dynamic array, random access
  • List: Frequent insertions/deletions at both ends
  • Deque: Insertions/deletions at both ends, fast random access
  • Set: Unique elements, sorted, fast lookup
  • Map: Key-value storage, sorted by key
  • Stack: LIFO operations (push, pop, top)
  • Queue: FIFO operations (enqueue, dequeue)
  • Priority Queue: Elements by priority, not FIFO

Vector

Dynamic array that grows/shrinks automatically. Most commonly used container.

#include <vector>
std::vector<int> v;
v.push_back(10); // Add element: O(1) amortized
v[0] = 5; // Access: O(1)
v.pop_back(); // Remove last: O(1)
v.erase(v.begin()); // Remove first: O(n)
v.size(); // Element count
v.capacity(); // Allocated size

Complexity: Access O(1), Insert O(n), Delete O(n)

Use when: Need fast random access, most general purpose

Deque (Double-Ended Queue)

Like vector but optimized for adding/removing from both ends.

#include <deque>
std::deque<int> d;
d.push_front(10); // Add to front: O(1)
d.push_back(20); // Add to back: O(1)
d.pop_front(); // Remove front: O(1)
d.pop_back(); // Remove back: O(1)
d[0]; // Random access: O(1)

Complexity: All operations O(1)

Use when: Need efficient operations on both ends

List (Linked List)

Doubly-linked list for efficient insertions/deletions anywhere.

#include <list>
std::list<int> l;
l.push_back(10); // Add: O(1)
l.push_front(5); // Add front: O(1)
l.insert(it, 15); // Insert at iterator: O(1)
l.erase(it); // Remove at iterator: O(1)
// No random access: l[0] NOT allowed!

Complexity: Insert/Delete O(1), Search O(n), No random access

Use when: Frequent insertions/deletions, don't need random access

Vector Basics & Operations

#include <vector>
using namespace std;

// Create and initialize
vector<int> v = {1, 2, 3};
vector<int> v2(5, 10); // 5 elements with value 10
vector<string> names;

// Basic operations O(1) or O(n)
v.push_back(4); // Add at end: O(1)
v.pop_back(); // Remove from end: O(1)
v.insert(v.begin()+2, 99); // Insert at position: O(n)
v.erase(v.begin()+1); // Remove at position: O(n)
v.clear(); // Remove all: O(n)
v.size(); // Get length
v.capacity(); // Memory allocated

Vector Memory Management

Dynamic Reallocation: When size exceeds capacity, vector allocates ~1.5x or 2x space

vector<int> v;
cout << "Initial capacity: " << v.capacity(); // 0

for(int i = 0; i < 10; i++) {
  v.push_back(i);
  if(i == 2 || i == 4) {
    cout << "Capacity: " << v.capacity() << endl;
  }
}
// Output: Capacity: 1, 2, 4, 8, 16...

💡 Tip: Use v.reserve(n) to pre-allocate space

2D Vector (Vector of Vectors)

// Matrix representation
vector<vector<int>> matrix = {
  {1, 2, 3},
  {4, 5, 6},
  {7, 8, 9}
};

// Access element at (1, 2)
cout << matrix[1][2]; // Output: 6

// Create 3x3 matrix initialized to 0
vector<vector<int>> grid(3, vector<int>(3, 0));

// Common operations

matrix.push_back({10, 11, 12}); // Add new row

Understanding Pair

#include <utility>

// Create pairs
pair<int, string> p = {1, "One"};
pair<string, double> student = make_pair("Alice", 95.5);

// Access elements
cout << p.first; // 1
cout << p.second; // "One"

// Pair comparison
auto p1 = make_pair(1, 2);
auto p2 = make_pair(1, 3);
if(p1 < p2) cout << "p1 is less"; // Compares first, then second

Vector of Pairs

// Store coordinates
vector<pair<int, int>> points = {{1,2}, {3,4}, {5,6}};

// Store student records (name, marks)
vector<pair<string, int>> students;
students.push_back({"Alice", 90});
students.push_back({"Bob", 85});

// Access
cout << students[0].first; // "Alice"
cout << students[0].second; // 90

// Iteration
for(auto & p : students) {
  cout << p.first << ": " << p.second << endl;
}

Map Container (Key-Value)

#include <map>

// Key-value pairs sorted by key - O(log n)
map<string, int> freq;

// Insert/Update
freq["apple"] = 5;
freq["banana"] = 3;
freq["apple"] = 7; // Updates existing

// Access
cout << freq["apple"]; // 7
cout << freq.count("grape"); // 0 (not found)

// Iteration (sorted by key)
for(auto & p : freq) {
  cout << p.first << ": " << p.second << endl;
}

Vector + Map Pattern

// Frequency counter
vector<int> arr = {1, 2, 2, 3, 3, 3, 4};
map<int, int> freq;

for(int x : arr) {
  freq[x]++; // Auto-initialized to 0
}

// Result: 1→1, 2→2, 3→3, 4→1

// Store pairs and sort
vector<pair<int, int>> result(freq.begin(), freq.end());
sort(result.begin(), result.end(),
  [](auto &a, auto &b){
    return a.second > b.second; // Sort by frequency
  });

Set

Sorted collection of unique elements. Usually implemented as Red-Black tree.

#include <set>
std::set<int> s;
s.insert(10); // Insert: O(log n)
s.erase(10); // Erase: O(log n)
s.find(5); // Search: O(log n)
s.size();
// Elements automatically sorted!
for(auto x : s) cout << x; // Prints in order

Complexity: Insert/Delete/Search O(log n)

Use when: Need sorted unique elements with fast lookup

Map

Sorted key-value store. Keys are unique and sorted.

#include <map>
std::map<std::string, int> m;
m["apple"] = 5; // Insert/Update: O(log n)
m.erase("apple"); // Erase: O(log n)
m.find("apple"); // Search for key: O(log n)
m.count("apple"); // Check if exists: O(log n)
for(auto p : m) { // Iterates in key order
  cout << p.first << p.second;
}

Complexity: All operations O(log n)

Use when: Need key-value storage with sorted access

Unordered Set / Unordered Map

Hash-based containers. Faster average case, no ordering.

#include <unordered_set>
#include <unordered_map>
std::unordered_set<int> us;
us.insert(10); // Average O(1), Worst O(n)
us.find(10); // Average O(1)
// No guaranteed order when iterating!

Complexity: O(1) average, O(n) worst case

Use when: Need fast lookup, don't care about order

Stack (LIFO)

Last-In-First-Out. Last element added is first removed.

#include <stack>
std::stack<int> stk;
stk.push(10); // Add to top: O(1)
stk.push(20);
stk.top(); // View top: O(1) - returns reference
stk.pop(); // Remove top: O(1)
stk.empty(); // Check if empty
stk.size();

Complexity: All operations O(1)

Use when: DFS, undo/redo, expression evaluation, backtracking

Queue (FIFO)

First-In-First-Out. First element added is first removed.

#include <queue>
std::queue<int> q;
q.push(10); // Enqueue to back: O(1)
q.front(); // View front: O(1)
q.pop(); // Dequeue from front: O(1)
q.back(); // View back element
q.empty();
q.size();

Complexity: All operations O(1)

Use when: BFS, scheduling, buffering, task queue

Priority Queue

Elements ordered by priority, not FIFO. Usually max-heap by default.

#include <queue>
std::priority_queue<int> pq; // Max heap
std::priority_queue<int,
  vector<int>,
  greater<int>> pq; // Min heap
pq.push(10); // Insert: O(log n)
pq.top(); // Highest priority: O(1)
pq.pop(); // Remove top: O(log n)

Complexity: Push O(log n), Pop O(log n), Top O(1)

Use when: Dijkstra's, Huffman coding, heap sort, best-first search
