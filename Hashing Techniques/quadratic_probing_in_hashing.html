Quadratic Probing in Hashing

Quadratic probing handles collisions by jumping using square offsets instead of checking the very next slot. That spreads probe positions out more than linear probing and reduces primary clustering.

Open intro page Next: double hashing

Concept

In quadratic probing, we still keep all keys inside the table array, but the next position is not just index + 1. Instead, the probe distance grows as 1^2, 2^2, 3^2, and so on. That means the search jumps farther away from the home slot after each collision.

Probe rule: index = (hash(key) + i * i) % tableSize, where i = 0, 1, 2, ...

Why It Helps

  • It breaks the straight next-slot pattern used by linear probing.
  • That usually reduces long runs of consecutive occupied slots.
  • It still uses only the array, so no linked nodes are needed.

Search Behavior

  • Search must follow the exact same quadratic jump pattern used during insertion.
  • It checks the home slot first, then the slots reached by square offsets.
  • If an empty slot appears before the key is found, the search can stop.

Main Tradeoff

Good

Less primary clustering than linear probing, so insertions often spread out better.

Limit

Quadratic probing may not visit every slot for every table size, so the choice of table size matters.

Pseudo Logic

1
for i from 0 to tableSize - 1
2
index = (hash(key) + i * i) % tableSize
3
if table[index] is empty
4
table[index] = key
5
stop
6
report overflow if no suitable slot is found

C Code Example

This version uses square offsets for the probe sequence.

#include <stdio.h>

#define TABLE_SIZE 7

int table[TABLE_SIZE];

void initTable() {
    for (int i = 0; i < TABLE_SIZE; i++) {
        table[i] = -1;
    }
}

void insert(int key) {
    int home = key % TABLE_SIZE;

    for (int i = 0; i < TABLE_SIZE; i++) {
        int probe = (home + i * i) % TABLE_SIZE;
        if (table[probe] == -1) {
            table[probe] = key;
            return;
        }
    }

    printf("No slot found for %d\n", key);
}

How To Read The Demo

  • The active key is highlighted on the left.
  • Yellow slots are checked and rejected because they were already occupied.
  • The orange slot is the final insertion position reached by quadratic probing.
  • The probe trail shows both the slot and the square jump that produced it.

What This Shows

  • The jump pattern is not consecutive like linear probing.
  • For a home slot of 4, the example visits 4, 5, 1, 6.
  • The jump to slot 1 comes from 4 + 2^2 = 8, and 8 % 7 = 1.
  • That modulo step is what makes the probing wrap back into the table range.
