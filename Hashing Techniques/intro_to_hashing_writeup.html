Intro to Hashing Writeup

Hashing is a technique that maps a key to a table index. That direct jump is what gives hash tables their fast average-time behavior for insert, search, and delete operations.

Open interactive visualizer

Core Idea

A hash function takes a key and computes a bucket position. Instead of scanning every stored element, the program goes to the computed bucket immediately. In a simple classroom example, the function h(k) = k % m is often used, where m is the table size.

Worked Example

If the table size is 7 and the key is 44, then 44 % 7 = 2. That means the key should be stored in bucket 2. If bucket 2 is empty, insertion is direct. If it already has another key, a collision occurs.

No collision

Key 18 goes to bucket 4 and the slot is empty, so insertion is immediate.

Collision

Key 32 also maps to bucket 4, so both keys must be stored under the same bucket.

Why Collisions Are Normal

  • A hash table usually has far fewer buckets than possible key values.
  • That means two different keys can easily compute the same bucket index.
  • Collisions do not mean hashing failed; they just mean the table needs a collision strategy.

Chaining

In chaining, each bucket can store multiple keys by attaching a small list or grouped chain to the bucket. This approach is straightforward to visualize and reason about, which is why it is commonly used for introductions to hashing.

Load Factor

The load factor is number of stored keys / table size. As the load factor grows, collisions become more frequent and chains become longer. That gradually pushes lookups away from ideal constant-time behavior.

Where Hashing Is Used

  • Hash maps and dictionaries
  • Sets and duplicate detection
  • Caches and memoization
  • Symbol tables in compilers and interpreters
