Open Addressing Strategy

Double Hashing in Hashing

Double hashing uses two hash functions. The first hash chooses the home slot, and the second hash decides how far to jump when a collision happens. That makes probe trails differ from key to key.

Concept

Double hashing is an open addressing method. Like other probing methods, it stores keys directly in the table array. The difference is that collision jumps are controlled by a second hash function instead of a fixed step or square offset.

Probe rule: index = (h1(key) + i * h2(key)) % tableSize
Demo functions: h1(k) = k % 7, h2(k) = 5 - (k % 5)

Why It Helps

  • Keys that share the same home slot can still follow different probe trails.
  • That reduces clustering more effectively than many simpler probing methods.
  • The second hash function must never produce 0, or probing will stop moving.

Search Behavior

  • Search computes both h1(key) and h2(key).
  • It then follows the same jump pattern used during insertion.
  • If an empty slot appears before the key is found, the search can stop.

Main Tradeoff

Good

Better distribution of probe trails, which usually means less clustering than linear probing.

Limit

The table size and second hash function must be chosen carefully so probing can reach enough slots.

Pseudo Logic

1
home = h1(key)
2
step = h2(key)
3
for i from 0 to tableSize - 1
4
index = (home + i * step) % tableSize
5
if table[index] is empty, place the key there
6
report overflow if no slot is found

C Code Example

This sample uses a second hash function to compute the jump size.

#include <stdio.h>

#define TABLE_SIZE 7

int table[TABLE_SIZE];

int h1(int key) {
    return key % TABLE_SIZE;
}

int h2(int key) {
    return 5 - (key % 5);
}

void initTable() {
    for (int i = 0; i < TABLE_SIZE; i++) {
        table[i] = -1;
    }
}

void insert(int key) {
    int home = h1(key);
    int step = h2(key);

    for (int i = 0; i < TABLE_SIZE; i++) {
        int probe = (home + i * step) % TABLE_SIZE;
        if (table[probe] == -1) {
            table[probe] = key;
            return;
        }
    }

    printf("No slot found for %d\n", key);
}

How To Read The Demo

  • The active key is highlighted on the left.
  • Orange slots show collisions where probing had to continue.
  • The green slot is the final insertion position for the active key.
  • Each chip in the probe trail shows the second-hash step used for that key.

What This Shows

  • All example keys have the same home slot because k % 7 = 4.
  • But their jump sizes are different because h2(key) is different.
  • That means collisions do not force every key into the same next-slot pattern.
  • This is the main reason double hashing is usually stronger than linear probing.
