Open Addressing Strategy

Linear Probing in Hashing

Linear probing handles collisions by checking the next slot, then the next one after that, until it finds an empty position. Instead of storing a chain, the table keeps all keys directly inside the array.

Open intro page Next: quadratic probing

Concept

In linear probing, a collision does not create a linked list. Instead, the algorithm moves one slot forward at a time until it reaches a free place. This is called probing. If it reaches the end of the table, it wraps around to the beginning.

Probe rule: index = (hash(key) + i) % tableSize, where i = 0, 1, 2, ...

Why It Helps

  • No extra linked nodes are needed because everything stays in the table array.
  • Insertion and search follow the same probe trail, which makes the logic consistent.
  • It is memory-friendly, but clustering can become a problem when many nearby slots get filled.

What Search Does

  • Start from the key's home slot using the hash function.
  • If that slot contains a different key, move to the next slot.
  • Continue until the key is found or an empty slot appears.
  • That is why insertion and search must follow the exact same probing rule.

Main Tradeoff

Good

Simple array-only structure, no extra pointers, and good cache locality.

Limit

Nearby occupied slots form clusters, so later insertions may scan many positions before finding space.

Pseudo Logic

1
for i from 0 to tableSize - 1
2
index = (hash(key) + i) % tableSize
3
if table[index] is empty
4
table[index] = key
5
stop
6
report overflow if no slot is found

C Code Example

This example stores keys directly in the array and uses -1 for an empty slot.

#include <stdio.h>

#define TABLE_SIZE 7

int table[TABLE_SIZE];

void initTable() {
    for (int i = 0; i < TABLE_SIZE; i++) {
        table[i] = -1;
    }
}

void insert(int key) {
    int index = key % TABLE_SIZE;

    for (int i = 0; i < TABLE_SIZE; i++) {
        int probe = (index + i) % TABLE_SIZE;
        if (table[probe] == -1) {
            table[probe] = key;
            return;
        }
    }

    printf("Hash table is full\n");
}

void display() {
    for (int i = 0; i < TABLE_SIZE; i++) {
        printf("Slot %d: %d\n", i, table[i]);
    }
}

What The Demo Proves

  • All four example keys have the same home slot.
  • Linear probing resolves each collision by checking the next slot.
  • The last example wraps from slot 6 back to slot 0.
  • This wrap-around behavior is an important part of open addressing.
