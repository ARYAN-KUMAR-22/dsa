LL Case

⬅️ Right Rotation

Triggered when a node has BF = +2 and its left child has BF ≥ 0. The left child pivots up.

  • Path: A → B → C (all left)
  • Fix: Single right rotation at A
RR Case

➡️ Left Rotation

Triggered when a node has BF = -2 and its right child has BF ≤ 0. The right child pivots up.

  • Path: A → B → C (all right)
  • Fix: Single left rotation at A
LR Case

↙️↗️ Left-Right Rotation

Triggered when BF = +2 but left child has BF = -1. Requires a double rotation.

  • Step 1: Left rotate on B
  • Step 2: Right rotate on A
RL Case

↘️↖️ Right-Left Rotation

Triggered when BF = -2 but right child has BF = +1. Requires a double rotation.

  • Step 1: Right rotate on B
  • Step 2: Left rotate on A

📐 Given Height h → Find Nodes

  • Max Nodes: Perfect tree → N_max = 2^h - 1
  • Min Nodes: Fibonacci/sparse AVL → use the recurrence:
    N(h) = N(h-1) + N(h-2) + 1

📐 Given Nodes n → Find Height

  • Min Height: Packed tree → h_min = ⌊log₂(n+1)⌋
  • Max Height: Sparse AVL → reverse lookup or h_max ≈ 1.44·log₂(n)

🏆 When to use AVL?

  • Read-heavy workloads (more searches)
  • When strict guaranteed height matters
  • Databases requiring fast lookups (e.g., SQLite uses B-trees, which are generalizations)

🔑 Why Height is O(log n)?

The minimum nodes in an AVL tree of height h follows the Fibonacci sequence. For large h:

N(h) ≈ φʰ / √5

Solving for h: h ≈ log_φ(n·√5) ≈ 1.44·log₂(n)

🌍 Real World Applications

  • Database indexing (balanced lookup trees)
  • Virtual memory management in OS
  • Computational geometry (interval trees)
  • Priority queues (heap alternatives)
  • Network routers (fast lookup tables)
