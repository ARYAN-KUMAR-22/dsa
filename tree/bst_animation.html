⚠️ Degeneration Problem

If you insert sorted data into a BST, it becomes a linked list. Height = n, search = O(n). This is the worst case AVL fixes.

💡 Solution

Use a self-balancing BST like AVL or Red-Black trees. They guarantee height = O(log n) regardless of insertion order.

📋 BST Property

  • Left child < Parent < Right child
  • All left subtree values < node
  • All right subtree values > node
  • Inorder traversal gives sorted output

⏱️ Complexity

Op Avg Worst
Search O(log n) O(n)
Insert O(log n) O(n)
Delete O(log n) O(n)

🗑️ Deletion Cases

  • Leaf: Just remove
  • 1 child: Replace with child
  • 2 children: Replace with in-order successor (smallest in right subtree)

🌍 Uses

  • Sorted data structures
  • Range queries
  • Symbol tables (compilers)
  • Base for AVL, Red-Black trees
