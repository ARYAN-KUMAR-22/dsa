Current Insight

DFS keeps moving deeper along one path until it cannot continue, then it backtracks to the most recent unfinished vertex.

What DFS Does

  • DFS starts from a source vertex and keeps going deeper whenever it finds an unvisited neighbor.
  • Only when it gets stuck does it backtrack to the previous vertex.
  • This creates a depth-first traversal order instead of a level-order one.

Why The Stack Matters

  • Recursive DFS uses the call stack, while iterative DFS uses an explicit stack.
  • The most recently discovered unfinished vertex is processed next.
  • This last-in, first-out behavior is what drives the search deeper before backtracking.

Where DFS Is Used

  • Cycle detection and connected component exploration.
  • Topological sorting ideas in directed acyclic graphs.
  • Backtracking-style problems such as mazes and puzzles.
  • Bridge, articulation point, and strongly connected component algorithms.

Complexity

  • With an adjacency list, DFS runs in O(V + E).
  • Each vertex is discovered once.
  • Each edge is inspected at most twice in an undirected graph.

C-Style Pseudo Code

dfs(node)
    visited[node] = true
    print node

    for each neighbor of node
        if neighbor is not visited
            dfs(neighbor)

C Code Sketch

void dfs(Node* graph[], int node, int visited[]) {
    visited[node] = 1;
    printf("%d ", node);

    Node* temp = graph[node];
    while (temp != NULL) {
        int next = temp->vertex;
        if (!visited[next]) {
            dfs(graph, next, visited);
        }
        temp = temp->next;
    }
}

Teaching Note

DFS marks a vertex as visited the moment it enters the search. That prevents the traversal from looping forever on cycles.
