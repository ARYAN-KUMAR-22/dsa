Current Insight

An undirected edge must update both endpoints, so every connection appears in both directions in the list and matrix.

What Undirected Means

  • An undirected edge has no arrow, so the connection works both ways.
  • If A connects to B, then B also connects to A.
  • This is why the matrix is symmetric and both adjacency lists are updated.

Adjacency List

  • Store the neighbors of each vertex.
  • Best for sparse graphs.
  • Space is usually O(V + E).

Adjacency Matrix

  • Use a V x V table of 0 and 1.
  • Edge lookup is fast.
  • Space is O(V^2).

Edge List

  • Store each undirected edge once as a pair like (u, v).
  • Simple to read and useful in many graph algorithms.
  • Not as fast as a matrix for direct edge checking.

C Code: Adjacency Matrix

#define V 6

void addEdge(int matrix[V][V], int u, int v) {
    matrix[u][v] = 1;
    matrix[v][u] = 1;
}

C Code: Adjacency List

void addEdge(Node* graph[], int u, int v) {
    addNode(&graph[u], v);
    addNode(&graph[v], u);
}

DSA Idea

All three forms store the same graph. They only differ in how the data is arranged and which operations become faster.
