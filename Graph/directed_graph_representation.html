Current Insight

A directed edge updates one source row, one matrix cell, and one arrow in the drawing.

What Directed Means

  • A directed edge has an arrow, so A -> B does not imply B -> A.
  • The source and destination play different roles in storage and traversal.
  • That is why directed matrices do not have to stay symmetric.

Outgoing List Effect

  • Adding (u, v) places v only in the outgoing list of u.
  • This keeps the arrow direction visible in a compact structure.
  • The active row in the animation shows which source vertex changed.

Adjacency Matrix Effect

  • matrix[u][v] = 1 means the arrow goes from u to v.
  • You do not automatically fill matrix[v][u].
  • This is why direction matters for reachability and flow problems.

In-Degree and Out-Degree

  • Out-degree: how many arrows leave a vertex.
  • In-degree: how many arrows enter a vertex.
  • The labels under each node help you compare both as the sample grows.

Execution Log

The theory view keeps a full log of the one-way updates so you can compare the active arrow with its list and matrix changes.

Pseudocode

for each directed edge (u, v):
    draw the arrow from u to v
    outgoing[u].append(v)
    matrix[u][v] = 1
    edgeList.append((u, v))
    indegree[v] += 1

C Code: Adjacency Matrix

#define V 6

void addEdge(int matrix[V][V], int u, int v) {
    matrix[u][v] = 1;
}

C Code: Adjacency List

void addEdge(Node* graph[], int u, int v) {
    addNode(&graph[u], v);
}

Storage Idea

Directed graph storage must preserve direction. That one-way rule is the key difference from the undirected case and changes how algorithms interpret every edge.
