Current Insight

BFS visits the graph level by level, so the queue always stores the frontier of vertices waiting to be processed.

What BFS Does

  • BFS starts from a source vertex and explores all neighbors first.
  • Then it moves to the next layer of neighbors, which is why it is called breadth first.
  • The queue guarantees vertices are processed in discovery order.

Why The Queue Matters

  • When a new vertex is discovered, it is pushed to the back of the queue.
  • The oldest waiting vertex is removed from the front.
  • This first-in, first-out behavior creates level-by-level traversal.

Where BFS Is Used

  • Shortest path in unweighted graphs.
  • Level-order exploration.
  • Connected component discovery.
  • Web crawling and network broadcasting style problems.

Complexity

  • With an adjacency list, BFS runs in O(V + E).
  • Each vertex is discovered once.
  • Each edge is inspected at most twice in an undirected graph.

C-Style Pseudo Code

visited[start] = true
enqueue(start)

while queue is not empty
    node = dequeue()
    print node

    for each neighbor of node
        if neighbor is not visited
            visited[neighbor] = true
            enqueue(neighbor)

C Code Sketch

void bfs(Node* graph[], int start, int V) {
    int visited[100] = {0};
    int queue[100], front = 0, rear = 0;

    visited[start] = 1;
    queue[rear++] = start;

    while (front < rear) {
        int node = queue[front++];
        printf("%d ", node);

        Node* temp = graph[node];
        while (temp != NULL) {
            int next = temp->vertex;
            if (!visited[next]) {
                visited[next] = 1;
                queue[rear++] = next;
            }
            temp = temp->next;
        }
    }
}

Step Mapping

The animation follows the same rhythm every time: initialize the start vertex, dequeue the front item, inspect neighbors, enqueue every undiscovered neighbor, and stop when the queue becomes empty.

Teaching Note

BFS marks vertices when they are enqueued, not when they are dequeued. That prevents the same vertex from entering the queue multiple times.
